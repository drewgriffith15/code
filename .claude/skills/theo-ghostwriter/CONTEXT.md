# THEO Pipeline — Domain Glossary

## Terms

### Source Material
The raw transcript downloaded from an external source (YouTube, podcast, etc.). Saved locally by Drew before any pipeline runs. Never generated by the pipeline. Never pushed to Notion.

### Source Outline
The structured lesson framework generated FROM source material by `theo-outline`. Represents the author's original content and teaching structure — not Drew's adaptations. This is the shareable artifact. Pushed to Notion (Studies database). Sent to substitute teachers as-is.

### Adapted Outline
Drew's modified version of the Source Outline. Trimmed for audience, time constraints, and series continuity (dedup against recent lessons). Used as input to the ghost-writer. Local only — never pushed to Notion. Not sent to subs.

### Draft
AI-generated lesson prose built from the Adapted Outline by `theo-ghost-writer`. Single working file named `{outline_stem}_draft.md`, saved to `Construct/wiki/theology/lessons/`. Overwrites on each run. Not retained long-term.

### Final Draft / Lesson
The lesson as actually taught. Lives in `Construct/wiki/theology/lessons/`. Retained permanently. This is the record of what Drew's audience heard.

## Data Flow (one-way: local → Notion, never back)

Source Material (local) → Source Outline (local + Notion push via /theo-outline) → Adapted Outline (local only) → Draft (local only) → Final Draft/Lesson (local + Notion push via /theo-push)
